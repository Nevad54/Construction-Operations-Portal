# Portfolio Demo Flow

## Goal

Show that the app handles both sides of an operations platform:

- public lead capture and positioning
- internal admin workflow and operational follow-through

## Demo Sequence

1. Public Homepage
- Show the hero, expertise cards, and selected outcome cards
- Explain that the app is positioned as a neutral construction operations platform

2. Contact Workflow
- Open the contact page
- Explain the intake path: inquiry creation, role assignment, next follow-up scheduling

3. Projects View
- Show that the public project gallery is part of a broader workflow, not just a static marketing page
- Use the portfolio narrative block to explain delivery visibility

4. Admin Dashboard
- Show KPI cards and inquiry management
- Highlight owner, status, priority, and next follow-up date enforcement

5. Overdue Follow-up Queue
- Show overdue inquiry visibility
- Demonstrate quick actions: `Snooze 1 Day` and `Mark Resolved`

6. Release Discipline
- Show `npm run verify:release`
- Explain that the demo is backed by build and test verification, not just screenshots

## Talking Points

- This app is suitable for contractors, facilities operators, and project-based service teams.
- The public site and the admin side are designed as one funnel, not separate products.
- The system demonstrates role-aware access, operational accountability, and portfolio-safe demo data.
